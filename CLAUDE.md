# CLAUDE.md — Project Map

## Project: Good Work CV (react-project)

React + Vite portfolio site. Run with `npm run dev`. Build with `npm run build`.

---

## Stack

- **React 19** + **react-router-dom 7** (SPA, hash-free routing via `vercel.json`)
- **Vite 6** (no TypeScript, plain JSX)
- **GSAP 3** + **ScrollTrigger** — all animation logic
- **Lenis** — smooth scroll
- **lottie-web** — Lottie JSON animations
- **lz-string** — data compression (used in admin/share flow)

## Key Conventions

- No TypeScript — plain `.jsx` / `.js` throughout.
- CSS is split per page/section (site.css, work.css, admin.css) — not CSS modules.
- Animation logic lives in `src/animations/` and is consumed via hooks in `src/hooks/`.
- Project data flows from `scripts/` → `src/data/projects.extracted.json` → `SiteContext`.
- `legacy/` is archived reference code, never imported by the React app.

---

## DO NOT Read Unless Editing

- `node_modules/` — never read these
- `legacy/` — archived, read only if asked about old implementation
- `public/documents/*.json` — large Lottie files, read only if debugging animation
- `src/data/projects.extracted.json` — generated file, read only if debugging data
- `package-lock.json` — never read

## Quick Navigation

| Task | Start here |
|------|-----------|
| Add a new page | `src/App.jsx` + `src/pages/` |
| Change nav | `src/components/layout/Nav.jsx` |
| Edit home sections | `src/components/home/` |
| Edit animations | `src/animations/` + `src/hooks/` |
| Change global styles | `src/styles/site.css` or `src/index.css` |
| Change site data | `src/data/site.json` or run `scripts/build-site-json.mjs` |
| Add/update projects | `scripts/extract-projects.mjs` → regenerate `projects.extracted.json` |
| Shared state / loader | `src/context/SiteContext.jsx` |
